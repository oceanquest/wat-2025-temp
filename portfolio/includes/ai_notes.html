Prompt: (PROVIDED EXAMPLE) Optimise the css file so that the styles cascasde properly

Date: 26-09-2025

The original style sheet didn't cascade the style optimally. Codepilot was used to make a suggestion and refactor the CSS. As an example: 

/* Original CSS */
#html .card {
  background-color:#ebfffc ;
  border-radius: 4px;
  border: 1px solid #dbdbdb;
  color: #363636;
  font-size: 1rem;
  margin-bottom: 1.5rem;
  position: relative;
}
/* Shared card styles */
.card {
    border-radius: 4px;
    border: 1px solid #dbdbdb;
    color: #363636;
    font-size: 1rem;
    margin-bottom: 1.5rem;
    position: relative;
}
/* Refactored CSS suggested by AI */
#html .card { background-color: #ebfffc; }

The refactor also introduced "composes: page-title;" which is a conceptual way to show reuse. 

.page-title {
    font-family: "Montserrat", sans-serif;
    font-weight: 700;
    border-bottom: #222525 2px solid;
    font-size: 2.5em;
    color: #333;
    margin-bottom: -0.3em;
}

#html > h1 { composes: page-title; }

Prompt: (PROVIDED EXAMPLE) why does vscode say composes css(unknownProperties)

Date: 26-09-2025

AI includes compose in the css suggestion - on further examination, it appears that "composes" is not a standard CSS property, which is why VSCode flags it as unknown. AI suggested new CSS syntax and just add the class attribute to the HTML. The new css flags a warning as empty style rule is left. This is being kept should the need arise for future use. 

#html > h1 { /* page-title styles applied via class */ }

<section id="html">
    <h1 class="page-title">Hypertext Markup Language (HTML)</h1>
</section>
